Discord Bots

Revro maintains a suite of Discord bots that power the community server and provide tools for server owners. Here's a quick overview of each one.

The Bots​

Revro Server Builder Bot​

The Server Builder bot is the backbone of Revro's Discord automation feature. It connects your Discord server to the Revro platform and executes AI-generated server setup instructions automatically.

What it does:

  • Creates and configures channels, categories, roles, and permissions
  • Sets up welcome and goodbye messages
  • Configures autorole for new members
  • Executes server blueprints generated by the Revro AI chat

Requires: Bot must be added to your server, and the server must be connected via the Revro dashboard.

➡️ Server Builder Guide

Tickets Bot​

The Tickets Bot creates private support channels for users who need help, keeps a record of all conversations, and manages queue capacity automatically.

What it does:

  • Creates a private text channel when a user opens a ticket
  • Notifies moderators/helpers of new tickets
  • Saves full transcripts when tickets are closed
  • Auto-closes tickets when the queue fills up

➡️ Tickets Bot Guide

FAQ Bot​

The FAQ Bot answers common questions automatically by monitoring ticket channels and forum posts. It uses Gemini AI to match user questions against a knowledge base of pre-written answers.

What it does:

  • Monitors ticket channels and forum posts (not general chat)
  • Matches incoming questions against 38+ pre-written FAQs
  • Responds automatically with relevant answers
  • Admin slash commands to manage the FAQ knowledge base

Adding a Bot to Your Server​

The Server Builder Bot is added automatically when you connect a Discord server in the Revro dashboard. Follow the OAuth prompt to grant the necessary permissions.

For Tickets and FAQ bots — these currently power the official Revro community server. For questions about access, ask in the Discord server.

:::tip Community Server All three bots are active in the Revro Discord server. Join to see them in action before deploying to your own server. :::
